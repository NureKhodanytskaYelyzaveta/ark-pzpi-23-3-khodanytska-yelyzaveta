ВИКОРИСТАННЯ ШІ ПІД ЧАС РОЗРОБКИ

Під час виконання лабораторної роботи було використано інструмент штучного інтелекту ChatGPT:
- назва інструменту та версія: ChatGPT (модель GPT-5.3, план Go);
- мета використання: допомога у формуванні структури документа, уточненні формулювань та описі архітектурних рішень відповідно до вимог;
- внесок автора: отримані результати були перевірені, відредаговані та адаптовані відповідно до архітектури системи автоматизації бібліотеки.

1. Запит (prompt) для формування розділу 3.2 (AI інтеграція):
Опиши, як можна інтегрувати штучний інтелект у систему для автоматизації бібліотеки.
Система включає backend (Flask), веб-клієнт, майбутній мобільний додаток та IoT-клієнт (поштомат).
Потрібно описати:
- як саме ШІ можна застосувати в системі;
- які дані можна збирати для навчання моделей;
- як краще використовувати ці дані — через готові API чи власні моделі.
Відповідь зроби у формальному стилі для документа Vision & Scope.

Отримана відповідь:
3.2 Scope of Subsequent Releases
У наступних версіях системи планується інтеграція технологій штучного інтелекту з метою покращення взаємодії користувачів із системою та розширення її функціональності.
ШІ може бути використаний для:
- формування персоналізованих рекомендацій книг на основі історії читання;
- надання автоматичної підтримки користувачам через чат-бот;
- автоматичної категоризації нових книг при додаванні до каталогу.
Для реалізації цих функцій система може збирати:
- історію пошуку та перегляду книг;
- інформацію про видачу та повернення;
- взаємодію користувачів із каталогом.
Передбачається використання комбінованого підходу:
- готові API — для реалізації чат-бота;
- власні або прості моделі — для рекомендацій;
- донавчання існуючих моделей — для категоризації.

2. Запит (prompt) для формування розділу 3.3 (Limitations and Exclusions):
Допоможи сформулювати розділ Limitations and Exclusions для системи автоматизації бібліотеки.
Потрібно:
- визначити основні обмеження системи;
- описати функції, які не включено;
- пояснити причини.

Отримана відповідь:
3.3 Limitations and Exclusions
Поточна версія системи має низку обмежень, пов’язаних із архітектурою та обсягом першого релізу.
Обмеження системи:
- відсутність повноцінного мобільного додатку (на етапі планування);
- IoT-клієнт реалізовано у вигляді емулятора, а не фізичного пристрою;
- обмежена масштабованість (один сервер);
- залежність від стабільного інтернет-з’єднання.
Виключений функціонал:
- офлайн-режим роботи — через складність синхронізації;
- інтеграція з платіжними системами — через вимоги безпеки;
- розширені AI-функції — через недостатню кількість даних;
- підтримка декількох бібліотек (multi-tenant) — через складність архітектури.

Отримані відповіді були проаналізовані та відредаговані відповідно до архітектури системи.
У розділі 3.2 було залишено ключові ідеї використання штучного інтелекту, зокрема рекомендації книг, чат-бот і категоризацію. При цьому формулювання було спрощено та адаптовано під конкретну систему. У частині збирання даних було залишено лише ті типи даних, які реально можуть використовуватися, зокрема дії користувачів і інформацію про видачу книг. У розділі 3.3 було відібрано основні обмеження, що відповідають поточній реалізації системи, та спрощено їх опис.

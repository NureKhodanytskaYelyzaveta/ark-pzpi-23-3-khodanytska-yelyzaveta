3 ОПИС ВИКОНАНОЇ РОБОТИ 
У межах лабораторної роботи було проаналізовано попередню версію документу Vision & Scope та внесено зміни відповідно до архітектурних вимог. Основна увага приділялася розширенню розділу 3 «Scope and Limitations», який тепер містить детальний опис архітектури системи, план інтеграції штучного інтелекту та обґрунтовані обмеження функціоналу.
На початковому етапі було проведено аналіз існуючої системи на основі матеріалів попередніх лабораторних робіт. У результаті було визначено ключові компоненти системи: серверну частину на Python/Flask із REST API та використанням SQLite, веб-інтерфейс для бібліотекарів і адміністраторів, IoT-клієнт (поштомат) з OTP-автентифікацією, а також основні моделі даних (User, Book, Loan, Reservation) та їх взаємозв’язки. Отримані результати стали основою для подальшого доповнення та уточнення документа.
У розділі 3.1 (Scope of Initial Release) описано загальну архітектуру системи. Система розглядається не як окремий веб-додаток, а як єдина система, у якій сервер, веб-клієнт, майбутній мобільний додаток та IoT-пристрої взаємодіють між собою.
Також описано обмін даними між компонентами: клієнти надсилають запити через HTTP/REST, IoT-пристрої можуть використовувати MQTT для передачі даних у реальному часі, а всі частини системи працюють із спільною базою даних. Це забезпечує узгодженість даних і дає можливість легко розширювати систему в майбутньому.
Розділ 3.2 (Scope of Subsequent Releases) присвячено плану інтеграції штучного інтелекту. Було розглянуто три ключові аспекти. 
По-перше, застосування ШІ. Система може пропонувати персоналізовані рекомендації книг на основі історії читання, надавати автоматичну підтримку через чат-бот та автоматично категоризувати нові надходження. 
По-друге, збирання даних. Передбачається анонімізоване логування подій (пошук, перегляд, видача) зі згоди користувача, що дозволить формувати якісні датасети для навчання. 
По-третє, навчання та використання. Для рекомендацій планується власна легка модель на основі Collaborative Filtering, для чат-бота — використання готового API з промпт-інжинірингом, а для категоризації — fine-tuning існуючих NLP-моделей. Такий гібридний підхід дозволяє отримати максимум користі без надмірних витрат на розробку.
У розділі 3.3 (Limitations and Exclusions) систематизовано функціональні обмеження. Визначено функції, які були навмисно виключені через міркування безпеки, авторського права або технічної доцільності. Наприклад, пряма обробка платежів виключена на користь редіректу на зовнішні шлюзи, повнотекстовий пошук по книгам недоступний через авторські права, а розпізнавання пошкоджень книг через Computer Vision вимагає спеціалізованого обладнання. Кожне обмеження обґрунтовано, що допомагає уникнути розмивання вимог у майбутньому.

1. Business Requirements (Бізнес-Вимоги)
1.1  Background (Передумови)
У сучасному інформаційному просторі бібліотеки залишаються важливими центрами знань й освіти. Проте багато бібліотек досі покладаються на ручні процеси обліку фондів, видачі книг і взаємодії з читачами, що призводить до перевантаження персоналу, помилок у документообігу та незадовільного досвіду людей. Існуючі рішення часто є застарілими, складними або не враховують потреб сучасних читачів, які очікують швидкого і зручного доступу до послуг. 
Саме тому було вирішено розробити сучасну програмну систему для автоматизації роботи бібліотеки — інтегровану платформу, яка не лише спрощує внутрішні операції (облік фондів, видачу та повернення матеріалів, формування звітів), а й надає читачам зручний доступ до електронного каталогу, можливість онлайн-бронювання, продовження строку користування та отримання сповіщень про прострочені книги. Завдяки цьому рішенню бібліотека зможе ефективно підвищити якість обслуговування та залишатися актуальним культурно-освітнім простором у нашу епоху.
1.2  Business Opportunity (Бізнес-Можливості)
Ринок бібліотечного програмного забезпечення залишається недостатньо адаптованим до сучасних цифрових потреб користувачів і персоналу. Більшість існуючих рішень або є застарілими настільними системами з поганим інтерфейсом або надто складними та дорогими для невеликих бібліотек. Водночас сучасні читачі очікують зручного онлайн-доступу до каталогів, можливості самостійного бронювання та отримання сповіщень, чого традиційні системи не забезпечують. 
Програмна система для автоматизації роботи бібліотеки пропонує сучасне та інтуїтивно зрозуміле рішення. Це робить систему конкурентоспроможною серед сучасних бібліотечних платформ і привабливою для бібліотек. 
Цільова авдиторія системи: 
Бібліотекарі та адміністратори — для автоматизації рутинних операцій (видача/повернення, облік фондів, формування звітів); 
Читачі (користувачі бібліотеки) — для зручного пошуку книг, онлайн-бронювання, продовження строку користування та отримання сповіщень; 
Керівництво бібліотек — для аналітики та статистики бібліотеки на користувачів.
Аналоги на ринку та конкуренти: 
Open-library (https://openlibrary.org/) — некомерційна онлайн-бібліотека, що надає вільний доступ до мільйонів оцифрованих книг з усього світу, дозволяючи користувачам “позичати” та читати їх безкоштовно;
Wiley (https://onlinelibrary.wiley.com/) — велика академічна платформа, яка пропонує доступ за підпискою до понад 20 000 наукових журналів, тисяч книг та довідкових матеріалів, орієнтованих на дослідників, студентів і професіоналів;
BookNet (https://booknet.ua/) — українська літературна платформа для самвидаву, де сучасні автори публікують свої твори, а читачі можуть купувати до них доступ, читати безкоштовні уривки та спілкуватися з письменниками.
Проблеми, які вирішує система: 
Відсутність доступного, сучасного та зручного у використанні ПЗ для автоматизації роботи бібліотек; 
Обмежена взаємодія між бібліотекою та читачами через відсутність онлайн-сервісів; 
Неможливість швидкої генерації аналітичних звітів, що ускладнює прийняття управлінських рішень.
Завдяки гарному поєднанню простоти, функціональності та сучасного інтерфейсу програмна система для автоматизації роботи бібліотеки стане не просто інструментом обліку, а цифровою платформою, яка допоможе бібліотекам залишатися актуальними, ефективними та орієнтованими на користувача.
2. Vision Of The Solution (Концепція Рішення) 
2.1  Vision Statement (Окреслення Концепції)
Програмна система для автоматизації роботи бібліотеки — це сучасна цифрова платформа, яка об’єднує ефективне управління бібліотечними фондами з інтуїтивним зрозумілим інтерфейсом для читачів. Система спрощує повсякденні операції бібліотекарів, забезпечує швидкий доступ до книг для користувачів і перетворює бібліотеку на динамічний освітній простір. Платформа поєднує функції електронного каталогу, особистого кабінету читача та інструментів адміністрування, що робить її універсальним рішенням для бібліотек. 
Основна мета системи: 
Автоматизація бібліотечних процесів — бібліотекарі можуть швидко видавати та приймати книги, вести облік фондів, формувати звіти та керувати простроченнями за допомогою зручного інтерфейсу;
Зручний доступ для читачів — користувачі отримують можливість шукати книги в електронному каталозі, бронювати їх онлайн, продовжувати строк користування, отримувати сповіщення про повернення та переглядати історію своїх замовлень;
Аналітика та прийняття рішень — керівництво бібліотеки отримує інструменти для аналізу популярності, активності читачів та ефективності програми, що сприяє стратегічному розвитку закладу. 
Система орієнтована на бібліотекарів, керівників бібліотек, студентів, школярів та всіх любителів читання, які прагнуть зручного та сучасного доступу до знань. Головне бачення проекту — створити відкриту та зручну платформу, яка не лише оптимізує роботу бібліотеки, а й надихне людей читати й навчатися чомусь новому. 
2.3  Major Features (Головна Функціональність)
Програма надає основний функціонал для трьох видів користувачів: читача, бібліотекаря й адміністратора. Кожна роль має специфічний набір можливостей, завдяки якому система ефективно працює за своїм призначенням.
Читач: 
Зареєструватися (через email або Google) і увійти;
Шукати та переглядати книги;
Бронювати книги і продовжувати строк;
Бачити свої поточні та минулі книги. 
Бібліотекар:
Видавати та приймати книги;
Додавати, редагувати або видаляти книги в каталозі;
Переглядати список читачів і їхні замовлення.
Адміністратор:
Керувати всіма даними — користувачами, книгами, налаштуваннями;
Дивитися звіти — які книги популярні, хто активний тощо.
3.  Scope and Limitations (Рамки та Обмеження Проєкту)
3.1  Scope of Initial Release (Рамки Первинного Випуску)
Перший випуск системи охоплює повний архітектурний контур проєкту. Система побудована навколо центрального сервера на базі Flask, який виконує роль єдиного джерела істини для всіх даних. Сервер надає REST API, через який взаємодіють усі клієнтські компоненти: веб-інтерфейс для персоналу бібліотеки, майбутній мобільний додаток для читачів та IoT-клієнт у вигляді інтелектуального поштомата.
Веб-клієнт реалізований як адаптивний інтерфейс, що дозволяє бібліотекарям швидко оформлювати видачу та повернення книг, керувати каталогом, а адміністраторам — налаштовувати системні параметри та формувати звіти. Мобільний додаток надасть читачам можливість шукати книги, бронювати їх, переглядати історію позик та отримувати сповіщення. IoT-клієнт, реалізований як емулятор поштомата, дозволяє користувачам самостійно отримувати заброньовані книги за допомогою OTP-коду, що генерується сервером.
Взаємодія між компонентами здійснюється через стандартні протоколи: HTTP/REST для основних запитів та MQTT для подій реального часу від IoT-пристроїв. Така архітектура забезпечує незалежність клієнтів від сервера, простоту тестування та можливість поступового розширення функціоналу без порушення цілісності системи.

3.2  Scope of Subsequent Releases (Рамки Наступних Випусків)
У наступних версіях системи планується інтеграція технологій штучного інтелекту для підвищення її інтелектуальності та зручності для користувачів. План інтеграції розглядається через три ключові аспекти:
1. Застосування (Application):
Розумна система рекомендацій: Аналіз історії читання та уподобань для пропозиції релевантних книг (аналогічно до Netflix та Spotify);
Чат-бот підтримки: Автоматична відповідь на часті запитання читачів (графік роботи, правила, статус боргу) природною мовою;
Автоматична категоризація: ШІ-аналіз опису книги при додаванні в каталог для автоматичного присвоєння тегів та жанрів.
2. Збирання даних (Data Collection):
Джерела даних: Логи пошукових запитів, історія видачі книг, рейтинги та відгуки користувачів, час перегляду сторінки книги;
Метод збирання: Анонімізоване логування подій (event logging) на сервері. Користувачі мають можливість погодитися на збір даних;
Зберігання: Окреме сховище даних або розширена схема БД для зберігання метаданих поведінки користувачів.
3. Навчання/Використання (Training/Usage):
Рекомендації: Використання алгоритмів Collaborative Filtering та Content-Based Filtering. Модель донавчається періодично на накопичених даних бібліотеки;
Чат-бот: Використання готового API (наприклад, OpenAI або локальні open-source моделі) для обробки природної мови, щоб уникнути витрат на тренування з нуля;
Категоризація: Fine-tuning існуючих моделей класифікації тексту на датасеті бібліотечних анотацій.
3.3  Limitations and Exclusions (Обмеження та Винятки)
Незважаючи на широкий потенціал системи, деякі функції були навмисно виключені з архітектури через технічні, правові або економічні обмеження. Це дозволяє зосередитися на ключовому функціоналі та уникнути розмивання вимог. Нижче наведено перелік основних обмежень та причини їх введення.

4. Business Context (Бізнес-Контекст)
4.3.  Operating Environment (Робоче Середовище)
	Система базуватиметься на продуктивних і масштабованих технологіях, сумісних з IoT-пристроями.
Backend: використовуватиметься мова Python та фреймворк Flask для надійної обробки даних і швидкої розробки;
База даних: SQLite — легка вбудована СУБД для ефективного зберігання структурованої інформації;
Frontend: веб-інтерфейс створюватиметься з використанням HTML, CSS і JavaScript, що забезпечить сучасний та зручний користувацький досвід;
IoT-комунікація: взаємодія між пристроями (наприклад, сканером штрих-кодів або модулем ESP32) і сервером здійснюється через протокол HTTP або MQTT, що дозволить в реальному часі передавати дані про видачу та повернення книг до веб-додатку;
Система керування версіями: використовуватиметься Git із розміщенням проєкту на GitHub для роботи та контролю версії;
Середовище розробки: Visual Studio Code, що забезпечить зручне налагодження коду, інтеграцію з Git і розширення для роботи з Flask.
